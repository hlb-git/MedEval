Introduction

Welcome to the Medeval documentation. This API allows you to access data, perform operations, and integrate it with your application seamlessly.

Authentication

To access the API, include your API key in the request header:

curl -H "Authorization: Bearer YOUR_API_KEY" https://api.example.com/resource

Endpoints

Here are the available API endpoints:

GET /api/patients   - Fetch a list of all patients
GET /api/patientlogin     - Fetch user details by Email
POST /api/patientsignup   - Create a new user
PUT /api/patientupdate    - Update user details
DELETE /api/patient    - Delete a user

* The above structure is applicable to other API endpoints (e.g Doctor, Pharmacist)

Examples

Example of fetching all patients:

curl -X GET https://hlbdev.tech/api/patients

All data required to create patient:

curl -X POST https://api.example.com/users \
-H "Content-Type: application/json" \
-d '{
"firstname": [string],
"lastname": [string],
"email": [string],
"password": [string],
"DOB": [date],
"countryCode": [string],
"phoneNumber": [string],
"gender": [string],
"country": [string],
"height": [float],
"weight": [string],
"maritalStatus": [string],
"age": [int],
"nin": [string]
}'
    Note:
  •     All payload must come in the request body, NO params!

Error Handling

Errors are returned in JSON format. Example response:

{
  "error": {
    "code": 401,
    "message": "Unauthorized. Invalid API key."
  }
}
